동아리 보고서

1. api/client.js      → 모든 요청의 공통 설정
                     (토큰 자동 추가, 401 자동 로그아웃)
모든 서버 요청에서 공통으로 쓰는 axios 설정
집배원(axios)에게 "항상 이 도장(토큰) 찍어서 배달해"라고 미리 설정해두는 것
 401 에러 = "로그인 안 된 사람" 응답이 오면 자동 로그아웃(토큰 삭제, 로그인 페이지 이동)

2. api/authApi.js     → 로그인·회원가입·내 정보 조회 함수 제작
                     → 예지가 사용
로그인 함수
1. 서버 응답 대기
2. 서버 내 저장된 토큰과 모든 유저정보를 브라우저에 저장
#실제 활용에서는 브러우저에 저장된 정보와 
로그아웃 함수
1.토큰 삭제
2. 로그인 페이지 이동
회원가입 함수
이름, 이메일, 비밀번호 전달
내 정보 조희 함수
토큰을 보내면 서버가 유저 정보 돌려줌

3. api/userApi.js     → 친구·채팅방 관련 요청
                     → 서현이 사용
검색어로 유저 찾기 (서현의 SearchUser 컴포넌트에서 사용)
?q=검색어 = URL에 검색 조건 붙이는 방식 (쿼리 파라미터라고 함)
ex) `/users/search?q=${query}`
내 친구 목록 가져오기
 친구 추가 (userId = 추가할 상대방의 고유 번호)
내 채팅방 목록 가져오기
채팅방 만들기 (이미 있으면 기존 방 반환)

4. api/messageApi.js  → 메시지 불러오기·보내기 함수
                     → 지후·예지가 사용
서버에 "데이터 줘"라고 요청하는 것

5. store/authSlice.js → 로그인 상태 창고
                     → 모든 팀원이 읽음
 로그인 성공 시 창고에 유저 정보와 토큰 저장
프로필 수정 후 창고의 유저 정보만 업데이트→ 서현의 ProfilePage에서 사용

6. store/messageSlice.js → 메시지 창고
                        → 지후가 addMessage로 씀
창고에 보관되는 데이터:
┌──────────────────┐
│  messages     : [ 메시지1, 메시지2, ... ] │  ← 메시지 배열
│  currentRoomId: "방 고유번호"           │  ← 지금 열린 채팅방
│  isLoading    : true / false                 │  ← 불러오는 중 여부
└────────────────
채팅방 들어갈 때 과거 메시지 전체를 한번에 저장
실시간으로 새 메시지 하나 추가→ 지후의 소켓 코드에서 사용 (★ 핵심!)
다른 채팅방으로 이동할 때 현재 방 ID 변경
채팅방 나갈 때 메시지 배열 비우기

7. App.jsx            → 앱 뼈대·라우팅·자동로그인
                     → 팀 전체 영향
창고를 앱 전체에서 쓸 수 있게 감싸줌
라우팅 = URL 주소에 따라 어떤 페이지를 보여줄지 결정하는 것

주소창                                → 보여주는 페이지
/login                               → LoginPage   (누구나)
/                                    → ChatPage    (로그인한 사람만)
/friends                             → FriendsPage (로그인한 사람만)
/profile                             → ProfilePage (로그인한 사람만)
로그인 안 한 채로 / 접속 시 → 자동으로 /login으로 보냄
