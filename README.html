이 메신저의 특징
Node.js, npm, 서버 설치가 전혀 필요 없어요. 메신저.html 파일 하나를 브라우저에서 열면 바로 실행됩니다.
같은 컴퓨터의 여러 탭을 열어서 실제처럼 채팅할 수 있어요.

🚀 시작하기

1
파일 열기
탐색기에서 메신저.html 파일을 더블클릭하거나,
브라우저(Chrome, Edge 등)에 파일을 드래그&드롭해서 열어요.
2
이름 입력 후 입장
로그인 화면에서 내 이름을 입력하고 입장하기 버튼을 누르거나 Enter 키를 누르세요. 이름은 최대 10자까지 입력할 수 있어요.
3
탭을 여러 개 열어서 채팅하기
같은 파일을 탭 2~3개 더 열고, 각각 다른 이름으로 입장하면 실시간으로 서로 메시지를 주고받을 수 있어요.
Ctrl + T → 새 탭 열기 → 주소창에 같은 파일 경로 붙여넣기

왼쪽 사이드바에서 채팅방을 클릭하면 전환됩니다. 각 채팅방은 독립적으로 대화 기록을 저장해요.

🌐 전체 채팅
모든 팀원이 볼 수 있는 공용 채팅방이에요. 공지사항, 일정 등 전달에 사용해요.
📚 공부방
과제나 프로젝트 관련 이야기를 하는 방이에요. 코드 질문, 자료 공유 등에 활용해요.
🎮 자유채팅
공부와 관계없이 자유롭게 대화하는 공간이에요.

💾 대화 기록 관련

메시지는 어디에 저장되나요?
브라우저의 localStorage에 저장됩니다. 이건 브라우저 안에 있는 작은 저장소예요. 새로고침하거나 브라우저를 닫았다 열어도 대화 기록이 그대로 남아 있어요. 방마다 최근 200개 메시지까지 보관합니다.
⚠️ 대화 기록을 지우고 싶을 때
브라우저 개발자 도구를 열어서 localStorage를 직접 삭제하면 됩니다.

Chrome / Edge 기준:
F12 → Application 탭 → 왼쪽 Local Storage → 파일 경로 클릭 → 오른쪽에서 항목 선택 후 Delete 키

⚠️ 알아둘 점

같은 컴퓨터 안에서만 작동해요
이 메신저는 BroadcastChannel 기술을 사용하는데, 이건 같은 컴퓨터의 같은 브라우저 탭끼리만 통신할 수 있어요. 친구 컴퓨터와 채팅하려면 가이드에 나온 것처럼 Node.js 서버(Socket.IO)가 필요해요.
기능 이 메신저 (지금) 실제 메신저 (가이드 목표)
통신 방식 BroadcastChannel (같은 컴퓨터 탭끼리) Socket.IO (인터넷으로 어디서나)
설치 필요 없음 — 파일 하나만 열면 됨 Node.js, npm, MongoDB 필요
메시지 저장 localStorage (내 브라우저에만) MongoDB (서버 DB에 영구 저장)
로그인 이름만 입력 (비밀번호 없음) 이메일 + 비밀번호 + JWT 토큰
다른 컴퓨터와 채팅 불가 가능 (서버 주소만 있으면)

⚙️ JavaScript 함수 목록

<script> 안에 있는 함수들의 역할을 정리했어요. 함수는 파이썬의 def 함수명():과 같은 개념입니다.

login()
이름 입력값 검증 → sessionStorage에 저장 → 화면 전환 → BroadcastChannel로 접속 알림
sendMessage()
입력창 텍스트 읽기 → renderMessage()로 내 화면에 표시 → saveMessage()로 저장 → channel.postMessage()로 다른 탭에 전송
renderMessage(msg)
말풍선 DOM 요소 생성 → isMine 값에 따라 클래스 결정 → chat-messages에 appendChild → scrollToBottom()
saveMessage(roomId, msg)
localStorage에서 기존 메시지 배열 꺼내기 → 새 메시지 push → slice(-200)으로 최근 200개 유지 → 다시 저장
loadMessages(roomId)
localStorage에서 해당 방 메시지 꺼내기 → forEach로 각 메시지를 renderMessage() 호출 → 사이드바 미리보기 업데이트
enterRoom(roomId)
currentRoom 변수 변경 → 헤더 텍스트 업데이트 → 사이드바 active 클래스 이동 → 채팅창 비우고 loadMessages()
renderOnlineUsers()
onlineUsers Set → 배열 변환 → 정렬 → forEach로 DOM 항목 생성 → online-list 업데이트
scrollToBottom()
chat-messages.scrollTop = scrollHeight 로 맨 아래 스크롤
updateLastMessage()
사이드바의 채팅방 미리보기 텍스트 업데이트. 20자 넘으면 "…" 붙이기
handleKeydown(event)
Enter 단독 → sendMessage(). Shift+Enter → 줄바꿈(기본 동작 허용)
autoResize(textarea)
입력창 높이를 40px로 초기화 후 scrollHeight로 다시 계산. 최대 120px 제한
escapeHTML(str)
innerHTML에 사용자 입력을 넣을 때 <script> 같은 특수문자를 무력화. XSS 보안 처리
toggleExplanation()
explanation-panel에 open 클래스를 toggle. CSS에서 open일 때 width: 340px로 전환
addDateDivider()
오늘 날짜 구분선("오늘 · 2025.05.23") DOM 요소를 만들어 채팅창에 추가

🚀 실제 서버로 확장하는 방법

BroadcastChannel을 Socket.IO로 바꾸면 다른 컴퓨터와도 채팅할 수 있는 진짜 메신저가 돼요. 코드 구조는 거의 그대로고 통신 방법만 바뀝니다.

지금 (BroadcastChannel)
// 채널 열기 const ch = new BroadcastChannel('app'); // 전송 ch.postMessage(data); // 수신 ch.onmessage = (e) => { handle(e.data); };
목표 (Socket.IO)
// 서버에 연결 const socket = io('http://서버:5000'); // 전송 socket.emit('chat', data); // 수신 socket.on('chat', (data) => { handle(data); });
이 메신저가 팀 프로젝트(가이드)와 어떻게 연결되나요?

이 파일은 팀 프로젝트를 시작하기 전에 "이런 앱을 만들 거야"를 눈으로 확인하는 프로토타입이에요.

• 이 파일의 renderMessage() → 가이드의 예지(UI 담당)가 만드는 MessageBubble.jsx
• 이 파일의 BroadcastChannel → 가이드의 지후(실시간 채팅)가 만드는 Socket.IO
• 이 파일의 localStorage → 가이드의 기혁(백엔드)가 만드는 MongoDB
• 이 파일의 channel.postMessage() → 가이드의 영민(기능 연동)이 만드는 Axios API 호출
